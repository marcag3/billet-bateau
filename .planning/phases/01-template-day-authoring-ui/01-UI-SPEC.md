## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline icon gaps, dense controls (`q-gutter-x-xs`, chip gaps) |
| sm | 8px | Compact spacing inside cards / list items (`q-gutter-sm`) |
| md | 16px | Default form rhythm and section spacing (`q-gutter-y-md`) |
| lg | 24px | Page wrap padding (`.page-wrap { padding: 1.5rem; }`) |
| xl | 32px | Large section separation (between major cards) |
| 2xl | 48px | Major section breaks on long edit screens |
| 3xl | 64px | Reserved for rare “page within page” separation only |

Wide-screen intent: use `--app-page-padding-xl: 3rem` (48px) when the layout uses `q-pa-xl` (existing pattern in `AppEntity*PageLayout`). This maps to the standard `2xl` spacing scale (48px) and does not introduce out-of-scale values.

**Rules:**

## Copywriting Contract

Tone: concise, operator-focused, confidence-inspiring. Prefer verbs + nouns, avoid jargon. Follow existing patterns like “Something went wrong. Please try again.” and confirmations that include the entity name.

| Element | Copy |
|---------|------|
| Primary CTA | **Save template day** (edit/create screen) |
| Empty state heading | **No template days yet** |
| Empty state body | Create one to define a typical day of departures for this program. |
| Error state | Something went wrong. Please try again. |
| Destructive confirmation | **Delete template day**: Delete this template day? This cannot be undone. |

**Phase 1 screen-level copy (English source of truth):**
- **Template days index**
  - Title: **Template days**
  - Description: Define reusable “typical days” with full slot details.
  - Primary action: **New template day**
  - Empty (no templates): Heading **No template days yet**; Body **Create one to start building your typical schedule.**
  - Empty (filtered/search): Heading **No matches**; Body **Try clearing filters.**
- **Template day create**
  - Title: **Create template day**
  - Description: Choose a program day pattern and add departures (slots).
  - Primary action: **Create template day**
  - Success toast: **Template day created.**
- **Template day edit/details**
  - Title: **Edit template day**
  - Description: Update slot details; changes sync locally and to server when online.
  - Primary action: **Save template day**
  - Success toast: **Template day saved.**
- **Slots section**
  - Section title: **Slots**
  - Primary action: **Add slot**
  - Empty: **No slots yet. Add a slot to define departures.**
- **Slot editor (inline dialog or side panel)**
  - Title (create): **Add slot**
  - Title (edit): **Edit slot**
  - Primary action: **Save slot**
  - Secondary action: **Close without saving**
  - Success toast: **Slot saved.**

## Interaction Contract (Phase 1 specific)

### Page structure & navigation
- Use existing page scaffolding:
  - Index/list pages: `AppEntityIndexPageLayout` pattern (Quasar `q-page` + `q-gutter-y-md`)
  - Create/edit pages: `AppEntityCreatePageLayout` / `AppEntityEditPageLayout`
- Back navigation:
  - Use top “Back” button with icon `arrow_back` and label **Back** or **Back to template days** depending on context.

### Core states (must be implemented consistently)
- **Loading**: show `AppBootstrapGate` loading state; never show partially-initialized forms.
- **Saving**:
  - Disable primary CTA; show inline spinner on the button.
  - Keep form values visible (no full-page loading).
- **Saved**: show Quasar `Notify` toast with success copy above; do not require dismissal.
- **Validation errors**:
  - Field-level messages (vee-validate global config: validate on blur).
  - Do not show a generic toast for validation-only failures.
- **Unexpected errors**:
  - Show a negative `Notify` using the shared helper behavior (generic message: “Something went wrong. Please try again.”).

### Visual hierarchy (focal points)
- **Template days index**: the primary focal point is the **Template days list card**; the secondary focal point is the **New template day** primary action (top-right / top of page header). Keep filters/search visually subordinate (less contrast than the list header).
- **Template day edit/create**: the primary focal point is the **page title + Save/Create template day CTA**; the secondary focal point is the **Slots** section header + **Add slot** action. Individual slot rows/cards must prioritize the summary string **time · boat type · route · capacity**.

### Slot editing behavior
- Slot editing must be **fast and low-risk**:
  - Prefer **inline edit** in the template day screen (dialog or expandable card), not full navigation for each slot.
  - Always show a read-only summary row/card for each slot with: **time · boat type · route · capacity**.
- Slot ordering:
  - Slots must render in **ascending departure time**.
  - If departure times collide, keep stable ordering by creation time.

### Destructive actions and confirmations
- Any delete requires confirm dialog using existing `useConfirmDialog()` behavior:
  - Dialog must be **persistent** (cannot dismiss by clicking outside).
  - Confirm copy must include the entity label/name when available.
- Required confirmations:
  - Delete template day
  - Delete slot

### Accessibility and touch targets
- Minimum touch target: 44px for icon-only buttons (Quasar `round dense` still must meet this; prefer non-dense on mobile if needed).
- All icon buttons must have `aria-label`.
- Form controls must have visible labels (do not rely on placeholder-only labeling).
