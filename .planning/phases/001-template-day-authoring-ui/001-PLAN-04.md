<objective>
Harden the template day authoring flow with validation and edge-case handling (time normalization, offline/outbox feedback, and regression tests) so staff can rely on it in real operations.
</objective>

<must_haves>
- Consistent time format stored for `template_day_slots.departure_time` (avoid mixed `HH:MM` vs `HH:MM:SS`).
- UI prevents invalid capacities and invalid ticket-setup payloads.
- Regression tests cover new fields and common failure modes.
</must_haves>

<threat_model>
- Threat: subtle data corruption through inconsistent time normalization leading to duplicated/unsorted slots.
  Mitigation: normalize time at a single boundary (UI helper) + tests.
</threat_model>
