<objective>
Implement the template day editor screen and slot authoring dialog per `001-UI-SPEC.md`, including ticket setup constraints and internal notes in the same workflow.
</objective>

<must_haves>
- Slot authoring includes: departure time, capacity, boat type, water route, ticket policy (defaults/custom), allowed ticket types, min/max per booking, and internal notes.
- ticket_setup is serialized as a JSON object with a canonical v1 shape (defined below).
- Outbox error banner is displayed using the standard pattern.
</must_haves>

<threat_model>
- Threat: UI serializes a ticket_setup shape that backend rejects → silent failure/outbox errors.
  Mitigation: canonical ticket_setup shape + shared validation + explicit automated verifies.
</threat_model>
